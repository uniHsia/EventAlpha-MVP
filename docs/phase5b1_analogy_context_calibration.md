# Phase 5B.1 Analogy Context Calibration

Phase 5B.1 improves the usability of historical analogy retrieval before Phase 5C outcome comparison. It keeps the Phase 5B scoring weights unchanged, but adds strength labels, input-context diagnostics, low-score explanations, cleaner verification suggestions, and richer demos.

This phase remains fully offline. It does not call LLMs, does not use embeddings or vector databases, does not change ledger schema, and does not compare real outcomes.

## Why Query-Only Scores Can Look Low

The analogy score uses eight dimensions: event type, affected assets, entities, industries, tags, causal chain, query keywords, and region. A query-only input can only score on `query_keywords`, whose weight is intentionally limited. Therefore a top case can be relevant while still showing a low numeric score.

The retriever now adds this warning when context is sparse:

```text
Low score mainly reflects limited input context, not necessarily an irrelevant case.
```

## Strength Labels

Every `HistoricalAnalogy` now includes a `strength_label`:

- `strong`: score >= 0.60
- `moderate`: 0.35 <= score < 0.60
- `weak`: 0.15 <= score < 0.35
- `surface_only`: score < 0.15

The label is only a research explanation of analogy strength. It is not an investment signal.

## Input Context Completeness

`AnalogyInputContext` reports:

- `provided_dimensions`
- `missing_dimensions`
- `context_completeness_score`
- `context_label`
- `low_context_warning`

Labels are:

- `query-only`: only query keywords were provided.
- `partial`: fewer than four dimensions were provided.
- `multi-dimensional`: four or more dimensions were provided.

## Event-Type Verification Suggestions

Verification suggestions are now generated by event family:

- `ai_export_control`: official rules, licensing scope, GPU orders, cloud capex, semiconductor supply chain, domestic substitutes, equipment/EDA mapping.
- `rate_policy`: central-bank statements, yield curve, FX, inflation, employment, expected policy path.
- `geopolitical_conflict` with oil/shipping context: oil inventories, shipping rerouting, production disruption, freight rates, risk premium.
- `trade_tariff`: tariff lists, retaliation, importer/exporter exposure, cost pass-through, alternative supply chains.
- `cloud_ai_capex` and `technology_breakthrough`: capex, orders, GPU supply, data-center power, networking, cooling.

This prevents AI export-control analogies from receiving unrelated central-bank, yield-curve, or FX suggestions.

## Combined-Context Demo

Run:

```bash
python scripts/run_historical_analogy_demo.py --demo-current-ai-export
```

This builds a richer current-event context with query, event type, assets, entities, industries, tags, and causal-chain notes. The top analogy should remain the AI chip export-control seed case, but the score becomes more meaningful because multiple dimensions participate.

## Active Event Demo

Run:

```bash
python scripts/run_historical_analogy_demo.py --from-active-event 1
```

This reads the first active `TrackedEvent` from the lifecycle JSON store and runs historical analogy retrieval without fetching news or modifying the store. If no active event exists, the script prints a clear message and suggests running the lifecycle tracker first or using the combined-context demo.

## Why This Is Not Phase 5C

Phase 5B.1 still does not compare real historical outcomes or ledger review results. It only improves analogy interpretation and current-event context handling. Phase 5C can use this output as a better input for historical outcome comparison.

## Risk Notice

Historical analogies are research aids only. They do not provide investment advice, trading instructions, target prices, or buy/sell recommendations. Seed outcomes are illustrative examples and are not verified market returns.
